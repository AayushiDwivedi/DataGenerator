DataGenerator
is an automated tool designed to ease the pain of test creation and maintenance.

Download latest executable »

Get involved on GitHub


Increased Coverage

DataGenerator generates test data using combinatorial coverage techniques like "pairwise combinations" and graphical coverage techniques like "all paths".

Flexible Output

Leveraging state-of-the-art templating, DataGenerator can produce test input, expected results, and automation scripts in any textual format.

Decreased Manual Effort

DataGenerator automates test maintenance, allowing you to work smarter and respond to change faster in an Agile environment.

 
Quick Start
  1. Download the latest DataGenerator Download latest executable »

  2. Make sure you have the latest JRE/JDK installed on your machine. If not, download it from Oracle.

  3. Get this download » of a sample data generator project.

  4. Extract the ZIP file contents from the previous step.

  5. Copy the latest DataGenerator executable JAR from step 1 to the parent ZIP extraction folder created in step 4.

    6. At this point your ZIP extraction folder should have the following structure:



    The extraction folder contains the following folders and files:
    'branchspec' folder contains the Visio dependency models.
    'dataspec' folder contains the Excel parameter definitions.
    'template' folder contains the DataGenerator output templates.
    'dd.properties' file contains global parameters for the DataGenerator.

    'GC.bat' is a sample windows command to execute allpaths graph coverage data generation. (NOTE:This primarily uses the Visio models to generate allpaths.)
    'COMBOS.bat' is a sample windows command to execute pairwise data generation. (NOTE: This exclusively uses the Excel parameter definitions.)

  6. To generate data on the sample provided:
    1. Allpaths graph coverage data[primarily uses Visio models]: Execute GC.bat file from the command shell or execute executable jar from the zip extraction folder. 
      CMD /c C:\DT\GC.bat
      OR
      java -jar DataDynamo-1.0-SNAPSHOT.jar -dataspec dataspec -branchgraph branchspec -templates template -out out_allpaths -allPaths -config dd.properties

    2. Pairwise data[exclusively uses Excel parameter definitions]: Execute COMBOS.bat file from the command shell or execute executable jar from the zip extraction folder. 
      CMD /c C:\DT\COMBOS.bat
      OR
      java -jar DataDynamo-1.0-SNAPSHOT.jar -dataspec dataspec -templates template -out out_pairwise -pairwise -config dd.properties


    7. - 'logs' folder will contain DataGenerator application logs.
    - 'out_*' folder will contain your generated output.

 
Get Involved

   Contribute

  1. We encourage everyone who has an idea to fork the code, experiment and share their experiences with us through our Google Group.

  2. If you believe that you have a worthwhile contribution, please open an issue on GitHub and explain your idea.

  3. Once you have written your code, please make sure to sign off your work when you commit it. 

    git commit -s -m 'YOUR COMMIT DESCRIPTION'

    git commit --signoff -m 'YOUR COMMIT DESCRIPTION'

  4. Once you have completed the commit you can make a pull request referencing the initial issue created for the work.

   Report Bugs

    When you encounter bugs, please try to find out if the issue has already been reported by searching here. Once you identify that your issue is new, create a new ticket. Please make sure you provide a clear description of the issue along with a testable scenario.

    If you want to get involved in fixing bugs, first read the steps outlined above under "Contribute". Then, you can assign yourself to the bug as the owner and start developing your code. Once you complete your code, build and test it, please make a pull request for review. Once the review is complete, your code fix will be merged into master branch within the DataGenerator repository.

   Communicate

    If you want to get involved, you can start by contributing to the GitHub project or reach out to opensource@finra.org.

    If you have any questions or discussion topics, please post them on Google Groups
